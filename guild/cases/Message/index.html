MxMessage 消息提示 ​

常用于主动操作后的反馈提示。 与 Notification 的区别是后者更多用于系统级通知的被动提醒。

基础用法 ​

从顶部出现，3 秒后自动消失。

MxUI 注册了一个全局的 $MxMessage方法用于调用。 Message 可以接收一个字符串或一个 VNode 作为参数，它会被显示为正文内容。

Show Code
vue
<template>
  <mx-button :plain="true" @click="open">Show message</mx-button>
  <mx-button :plain="true" @click="openVn">VNode</mx-button>
</template>

<script lang="ts" setup>
import { h } from 'vue'
// import { MxMessage } from '@wxct/mx-ui'
import * as MxUI from '@wxct/mx-ui'

const { MxMessage } = MxUI

const open = () => {
  MxMessage('This is a message.')
}

const openVn = () => {
  MxMessage({
    message: h('p', { style: 'line-height: 1; font-size: 14px' }, [
      h('span', null, 'Message can be '),
      h('i', { style: 'color: teal' }, 'VNode'),
    ]),
  })
}
</script>

不同状态 ​

用来显示「成功、警告、消息、错误」类的操作反馈。

当需要自定义更多属性时，Message 也可以接收一个对象为参数。 比如，设置 type 字段可以定义不同的状态，默认为info。 此时正文内容以 message 的值传入。 同时，我们也为 Message 的各种 type 注册了方法，可以在不传入 type 字段的情况下像 open4 那样直接调用。

Show Code
vue
<template>
  <mx-button :plain="true" @click="open2">Success</mx-button>
  <mx-button :plain="true" @click="open3">Warning</mx-button>
  <mx-button :plain="true" @click="open1">Message</mx-button>
  <mx-button :plain="true" @click="open4">Error</mx-button>
</template>

<script lang="ts" setup>
// import { MxMessage } from '@wxct/mx-ui'
import * as MxUI from '@wxct/mx-ui'

const { MxMessage } = MxUI

const open1 = () => {
  MxMessage('This is a message.')
}
const open2 = () => {
  MxMessage({
    message: 'Congrats, this is a success message.',
    type: 'success',
  })
}
const open3 = () => {
  MxMessage({
    message: 'Warning, this is a warning message.',
    type: 'warning',
  })
}
const open4 = () => {
  MxMessage.error('Oops, this is a error message.')
}
</script>

使用 HTML 片段作为正文内容 ​

message 还支持使用 HTML 字符串作为正文内容。

将dangerouslyUseHTMLString属性设置为 true,message 就会被当作 HTML 片段处理。

Show Code
vue
<template>
  <mx-button :plain="true" @click="openHTML">Use HTML string</mx-button>
</template>

<script lang="ts" setup>
// import { MxMessage } from '@wxct/mx-ui'
import * as MxUI from '@wxct/mx-ui'

const { MxMessage } = MxUI

const openHTML = () => {
  MxMessage({
    dangerouslyUseHTMLString: true,
    message: '<strong>This is <i>HTML</i> string</strong>',
  })
}
</script>

全局方法 ​

MxUI 为 app.config.globalProperties 添加了全局方法 $MxMessage。 因此在 vue 实例中你可以使用当前页面中的调用方式调用 Message

单独引用 ​

js
import { MxMessage } from '@wxct/mx-ui'
1

此时调用方法为 MxMessage(options)。 我们也为每个 type 定义了各自的方法，如 MxMessage.success(options)。 并且可以调用 MxMessage.closeAll() 手动关闭所有实例。
